# FE-3: Dashboard UI

**Role**: Frontend Engineer
**Context**: A lightweight UI to serve as the face of the FastAPI server, allowing users to see running experiments and list past reports.

## Requirements
1. Single Page Application (SPA) or simple HTML served by FastAPI.
2. Use ReactJS via CDN and TailwindCSS.
3. Features:
   - List available Experiments.
   - Trigger a new Sweep.
   - Link to generated Reports.
4. Consume the API endpoints defined in BE-4.

## Acceptance Criteria
- [ ] Dashboard loads.
- [ ] Can trigger an experiment.
- [ ] Can view status of experiments.
